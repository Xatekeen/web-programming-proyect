# Final Presentation Guide

## 5-Minute PowerPoint Structure

1. Project Aim and Objectives
   - Project: Sejong University Webpage Reproduction
   - Purpose: practice building a complete responsive webpage based on a real university site
   - Target users: prospective students, current students, visitors, and faculty

2. Project Features and Completeness
   - Responsive header and mobile menu
   - Separate pages: Home, About, Academics, Admissions, Research, Life at Sejong, News, Contact, and Demo
   - JavaScript features: expanded sidebar sitemap, search dialog, news filters, form response, animated counters, back-to-top button
   - Current status: functional static multi-page web application
   - Planned but not completed: backend admissions storage and real university API integration

3. Challenges Encountered
   - Creating a layout that works on multiple screen sizes
   - Keeping visual style consistent across sections
   - Making JavaScript interactions simple but useful
   - Balancing reproduction accuracy with available time and assets

4. Learning Outcomes
   - Semantic HTML and accessible navigation
   - CSS Grid, Flexbox, responsive breakpoints, animations
   - DOM manipulation and event handling in JavaScript
   - Team planning, task division, and live demo preparation

5. Team Member Contributions
   - Member 1: HTML structure and page content
   - Member 2: CSS styling, responsive layout, visual polish
   - Member 3: JavaScript interactions and testing
   - Member 4: presentation slides, demo script, documentation

## 5-Minute Live Demo Flow

1. Open the homepage and explain the multi-page structure.
2. Navigate to About, Academics, Admissions, Research, Life, and News using the top header.
3. Open the sidebar sitemap and show the deeper sublinks based on the real Sejong structure.
4. Open search and search for "admissions" or "research".
5. Submit the admissions inquiry form with sample data.
6. Navigate to News and use the category filters.
7. Navigate to Research and show the animated counters.
8. Resize the browser to mobile width and show the sidebar menu.
9. Finish on the Demo page and explain remaining possible improvements.
